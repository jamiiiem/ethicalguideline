Section E: Transparency

Transparency in AI chatbots involve being open and clear about the chatbot’s operation, the data is uses, and the decisions it makes. This section lists the specific goals that need to be achieved to ensure the users understand the chatbot’s functions, processes, and limitations, to better foster trust in the chatbot. These goals include explainability, data usage transparency, algorithmic transparency, capability disclosure, and public documentation.

E5 Public Documentation

Definition:

Developers should provide detailed documentation and reports on the chatbot’s development processes and performance insights.

Techniques:

  • Developer Documentation: publish technical documentation introduces the chatbot’s architecture, API references, and development processes.
  • Version Histories: maintain a history of updates and release notes.
  • Performance Metrics: publish performance metrics to give users a clear picture of the chatbot’s performance.
  • Incident Reporting: document records of significant incidents and recovery actions.
