Section A: Privacy and Data Protection

AI chatbots that deal regularly with educational content should handle them and interact with users with the highest standards of privacy and data security...

A1 Data Anonymisation

Definition:

  • Data should be collected with efforts to encrypt or erase userâ€™s private, sensitive, and identifiable information.

Techniques:

  • Data Masking: Substitute identifiable data with randomized data using shuffling or manipulation.
  • Tokenisation: Convert sensitive data into tokens usable in the system without exposing original data.
  • Noise Addition: Introduce minor randomization to ensure identities cannot be traced back.
  • Data Generalisation: Reduce precision of sensitive info, grouping data into categories.

Considerations:

  • Developers should balance privacy and utility of data.

Examples:

  • Names: Converted to randomised strings. Generalise ages into groups like 15-18 or 18-21 years old.

A2 Data Minimisation

Definition:

  • Data collected and retained should be minimised to relevant and essential data.

Techniques:

  • Data Collection Policies: Only gather essential data.
  • Data Retention Policies: Define how long data will be stored before automatic deletion.
  • Data Purging: Implement regular deletion of unused data.

Examples:

  • Automatic Purging: Set a timeframe to delete old data.
