Section A: Privacy and Data Protection

AI chatbots that deal regularly with educational content should handle them and interact with users with the highest standards of privacy and data security. This section lists the specific goals that need to be achieved in order to ensure that the privacy and data protection is upheld to a reasonable standard. These goals include data anonymisation, data minimisation, informed consent, secure data storage and transmission, user control, compliance with laws, as well as privacy audits.

A4 Secure Data Storage and Transmission

Definition:

Data should be stored securely in the server and transferred securely between the user and the chatbot.

Techniques:

  • Encryption: data collected and stored in the chatbot’s database should be securely encrypted using advanced standards (e.g., AES-256). End-to-end encryption should be used to encrypt the messages between the chatbot and the user to prevent unauthorised access during transmission.
  • Access Control: personnel that are allowed to access the backend server of the chatbot. Role-based access control and identity authentication are suitable measures to adopt.

Examples:

  • AES-256 may be used for encrypting the data and messages stored and transferred. In addition, multi-factor authentication may be used to verify the user’s identity and the right to access.
